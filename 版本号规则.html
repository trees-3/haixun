版本号的格式为 X.Y.Z     递增的规则为：


• X 表示主版本号，当 API 的兼容性变化时，X 需递增。

• Y 表示次版本号，当增加功能时(不影响 API 的兼容性)，Y 需递增。

• Z 表示修订号，当做 Bug 修复时(不影响 API 的兼容性)，Z 需递增。



详细的规则如下：

• X, Y, Z 必须为非负整数，且不得包含前导零，必须按数值递增，如 1.9.0 -> 1.10.0 -> 1.11.0

• 0.Y.Z 的版本号用于 API 不稳定；1.0.0 用于稳定的 API。

• 当 API 的兼容性变化时，X 必须递增，Y 和 Z 同时设置为 0。

• 当新增功能(不影响 API 的兼容性)或者 API 被标记为反对时，Y 必须递增，同时 Z 设置为 0。

• 先行版本号格式为：X.Y.Z.[a-c][正整数]，如 1.0.0.a1，1.0.0.b99，1.0.0.c1000。

• 开发版本号格式为 X.Y.Z.dev[正整数]，如 1.0.1.dev4。

• 版本号的排序规则为依次比较主版本号、次版本号和修订号的数值，如 1.0.0 < 1.0.1 < 1.1.1 < 2.0.0。版本一经发布，不得修改其内容，任何修改必须在新版本发布！
